# What Changed and Why

| File                     | Change                      | What changed                                                                                                                                                                       | Why                                                                                                             |
|:-------------------------|:----------------------------|:-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|:----------------------------------------------------------------------------------------------------------------|
| api-rest-design.md       | Refactored and expanded     | Replaced RFC 7807 with RFC 9457; added async 202, bulk semantics, resource identity, media-type nuance, and compatibility-safe write rules; softened aesthetic-only absolutes.     | Original was strong on CRUD consistency but missing async/bulk behavior and was too rigid in a few style rules. |
| api-versioning.md        | Replaced completely         | Turned duplicate REST harness into an actual versioning and compatibility policy.                                                                                                  | Uploaded file duplicated REST rules instead of defining versioning.                                             |
| api-security.md          | Refactored to security-only | Removed overlap with security headers and REST errors; clarified authn/authz, token use, validation, SSRF, secrets, audit, abuse protection.                                       | Original mixed transport, body-limit, docs exposure, and generic API concerns.                                  |
| api-security-headers.md  | Tightened and deduplicated  | Kept header/CORS/cache-only policy; removed duplicated broader security concerns.                                                                                                  | Makes it MECE with api-security.md.                                                                             |
| api-rest-resilience.md   | Modernized and narrowed     | Kept timeout/retry/health/shutdown/load-shedding rules; aligned transient retry statuses and removed transport/header duplication.                                                 | Makes resilience operational instead of partly overlapping with API design and CORS.                            |
| api-logging.md           | Separated from telemetry    | Kept structured logging, redaction, correlation, and security-event logging; moved distributed tracing concerns to telemetry.                                                      | Original overlapped with trace metadata and telemetry lifecycle.                                                |
| api-telemetry.md         | Separated from logging      | Focused on OpenTelemetry signals, propagation, span rules, metrics, sampling, and log-trace correlation.                                                                           | Original was good, but needed a sharper boundary with logging.                                                  |
| api-testing.md           | Condensed and deduplicated  | Preserved pyramid, isolation, contract testing, fixtures, and coverage suppression without framework-specific mechanics.                                                           | Fastify-specific test mechanics belong in the Fastify harness.                                                  |
| fastify-api-blueprint.md | Restructured heavily        | Stopped restating general API policy; now maps general rules to Fastify-specific implementation: plugin order, TypeBox schemas, AJV config, hooks, inject tests, lifecycle wiring. | This was the largest source of duplication across the set.                                                      |
